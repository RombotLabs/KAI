01Was ist KAI?

KAI ist eine DSGVO-freundliche KI-Chat-Plattform, die speziell für den Einsatz an Schulen entwickelt wird.

02Wer entwickelt KAI?

Das Projekt wird von RombotLabs auf GitHub entwickelt und gepflegt, im Repository RombotLabs/KAI.

03Ist KAI Open Source?

Ja. Der Quellcode ist öffentlich auf GitHub einsehbar und steht unter der MIT-Lizenz.

04Unter welcher Lizenz steht KAI?

KAI wird unter der MIT-Lizenz veröffentlicht, die eine freie Nutzung, Anpassung und Weiterverbreitung erlaubt.

05Was bedeutet „DSGVO-freundlich" bei KAI?

Laut Projektbeschreibung ist KAI so konzipiert, dass es den Anforderungen der Datenschutz-Grundverordnung für den Schuleinsatz gerecht wird. Konkrete technische Details oder eine offizielle Zertifizierung dazu sind im Repository aktuell nicht dokumentiert.

06Aus welchen Komponenten besteht das Projekt?

Zu den zentralen Dateien gehören unter anderem:

  • app.py – die Hauptanwendung
  • ufm.py (in libs/) – eine Bibliothek zur Verwaltung von Nutzer-Chats und hochgeladenen Dateien; für jeden neuen Nutzer wird automatisch ein eigenes Verzeichnis mit dessen Chats angelegt
  • passwd_hasher.py – vermutlich zuständig für das sichere Hashen von Passwörtern
  • templates/ – Vorlagen für die Benutzeroberfläche
  • requirements.txt – Liste der benötigten Python-Pakete
07Wie kann ich zum Projekt beitragen?

Für Entwickler:innen gilt laut README: Wird ein neues Python-Modul per pip installiert, muss der Modulname am Ende der requirements.txt ergänzt werden.

08Gibt es eine ausführliche Dokumentation?

Im Repository ist aktuell nur eine kurze README mit grundlegenden Hinweisen für Entwickler:innen vorhanden. Eine ausführliche Setup- oder Nutzungsanleitung ist derzeit nicht dokumentiert.

09Wie viele Personen nutzen das Projekt bereits?

Der Star- und Fork-Zähler auf GitHub steht aktuell bei 0 – es handelt sich offenbar um ein noch junges bzw. internes Projekt in früher Entwicklungsphase.
